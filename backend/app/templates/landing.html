C 第 3 节 详图 DETAIL · 能力清单 第 3/5 张
引线 03.A

平台对外暴露 16 个 MCP 工具,分 5 个能力组。每个都是带结构化 I/O 的轻量 REST 包装,既支持人工 CLI 调用,也支持 Agent 调用。

没有黑魔法,没有框架偏见,没有锁定。只有数据 + 状态流转。

能力组 04 当前为内测(预计 2026 Q3 公开发布)。

G.01

组件登记 Component Registry

组件增删改查、版本历史、按 layer / category / asset 过滤、递归依赖树。

稳定
G.02

反馈循环 Feedback Loop

Bug / 需求 / 问题登记,带严重度、报告人、GitHub Issue 关联、决策审计链。

稳定
G.03

部署地图 Deployment Map

登记 host × env × component,实时部署时间线,按主机聚合。

稳定
G.04

健康监控 Health Monitor 5 维度

组件覆盖、分层平衡、反馈速率、部署新鲜度、资产率。

测试中
G.05

需求追溯 Requirement Traceability

REQ → 能力 → 组件 反向图,在发布前捕获冲突和缺口。

测试中
G.06

资产决策引擎 Asset Decision Engine

反馈堆积时:优化 / 新建 / 保留 / 重新审视。决策有记录,不会丢。

草案
D 第 4 节 剖面图 SECTION · 反馈循环 第 4/5 张
"

这个平台不是一个登记簿。它是一个反馈循环 —— 每个被复用的组件都被观察,每次观察都变成一个决策,每个决策都更新登记簿。

在循环中活不下去的组件被分叉;长大了的被重新打版本;腐朽了的被归档。

这是大部分"组件库"会跳过的那一步。我们不跳。
